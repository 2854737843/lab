# 答辩思路（提纲）

## 1 30秒开场
- 我本次毕设题目是《基于 Spring Boot + Vue 的课题组项目协同与周报管理系统》
- 解决的问题：课题组成员协作分散、周报提交审核效率低、过程缺少留痕可追溯

## 2 背景与意义（1分钟）
- 课题组日常：多项目、多任务、多学生
- 管理痛点：任务分配/推进/复盘困难，周报收集与审核麻烦
- 需要：一个集成化平台实现协作、周报与审计

## 3 需求与功能（1分钟）
- 登录认证（JWT）
- 多课题组（一个用户可加入多个课题组）
- 项目任务协作（看板 TODO/DOING/DONE）
- 周报（自然周）提交与导师审核
- 审计追溯（关键操作留痕，可查询）

## 4 系统架构（1分钟）
- 前端 Vue3 + Vite（SPA）
- 后端 Spring Boot REST API
- 安全：Spring Security + JWT
- DB：MySQL（Docker Compose）

## 5 核心设计亮点（1分钟）
- 角色权限（ADMIN/MENTOR/STUDENT）
- 组成员关系表 group_members 支持多课题组
- 周报唯一约束：同一组同一作者同一自然周只能提交一次
- 审计表 audit_logs：actor/entity/action/time/detail，实现可追溯

## 6 演示流程（3-5分钟）
1) 使用 mentor 登录
2) 创建课题组 + 添加 student
3) 进入 项目/任务协作：创建项目、创建任务、拖动/按钮改变状态
4) student 登录提交周报
5) mentor 登录审核（通过/驳回）
6) mentor 打开审计查询页面，展示上述操作全部有记录

## 7 测试与结果（1分钟）
- 功能测试用例覆盖登录、组管理、任务流转、周报审核、审计查询
- 系统能够稳定运行，满足课题组日常协作

## 8 总结与展望（30秒）
- 总结：实现了完整业务闭环（协作 + 周报 + 审计）
- 展望：附件上传、消息通知、统计报表、权限细化、移动端适配

## 常见提问准备
- 为什么用 JWT？无状态、易扩展、适合前后端分离
- 如何保证周报不重复？数据库唯一约束 + 接口校验
- 审计的意义？责任归属、过程留痕、便于复盘
