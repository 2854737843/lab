**Design and Implementation of a Research Group Collaboration and Weekly Report Management System Based on Spring Boot and Vue**

Author: ________

College, Class, Student ID: ________

【Abstract】
University research groups often require efficient management of members, collaborative task execution, weekly report submission and mentor review, and traceable records of key actions. Traditional approaches based on instant messaging and offline documents usually lead to scattered information, opaque task progress, high overhead in weekly report collection and feedback, and a lack of traceability. To address these issues, this thesis designs and implements a research group collaboration and weekly report management system based on Spring Boot and Vue. The system follows a front-end/back-end separation architecture. The back-end provides RESTful APIs built with Spring Boot and implements stateless authentication and role-based authorization using Spring Security and JWT. The persistence layer is implemented with JPA and MySQL, modeling core entities including users, groups, memberships, projects, tasks, weekly reports, and audit logs. The front-end is developed with Vue 3 and Vite as a single-page application that offers group management, Kanban-style task collaboration (TODO/DOING/DONE), weekly report submission and mentor review, and audit querying. Key business operations are recorded in audit logs to support traceability. Testing results show that the system can effectively support daily collaboration and weekly report management in research groups and provides good extensibility.

【Keywords】research group management; task collaboration; weekly report; Spring Boot; Vue; JWT; audit log

# 1 前言
## 1.1 研究背景
随着高校科研团队规模扩大，课题组内部常同时开展多个项目，导师需要对学生任务进度、阶段成果和科研过程进行持续跟踪与指导。现实中，任务分配与协作通常依赖微信群/QQ群等即时通讯工具，文档资料分散在个人电脑与网盘，缺少统一的协作入口；周报制度往往以邮件或表格形式提交，导师审核与反馈难以形成闭环，历史记录检索也不便。此外，科研过程管理越来越强调可追溯性与留痕，缺少审计机制会导致责任边界不清、复盘成本高。

## 1.2 研究意义
（1）对导师：统一成员与任务管理入口，提升过程监管能力；周报审核留痕，便于指导与考核。
（2）对学生：明确任务与优先级，减少沟通成本；周报提交规范化，形成自我总结与计划。
（3）对课题组：沉淀协作过程数据，支持复盘与审计，提高整体协作效率。

## 1.3 本文主要工作
本文完成了课题组项目协同与周报管理系统的需求分析、架构设计、数据库设计与功能实现。主要实现了：JWT 登录认证、多课题组管理、项目与任务协作（看板）、周报提交与导师审核、审计日志记录与查询，并给出了系统测试与部署说明。

# 2 相关技术与可行性分析
## 2.1 Spring Boot
Spring Boot 通过自动配置与约定优于配置降低了企业级应用开发复杂度。本文后端采用 Spring Boot 构建 REST API，配合 MVC 分层结构实现业务模块化。

## 2.2 Spring Security 与 JWT
Spring Security 提供成熟的认证授权框架。本文使用 JWT 作为令牌，实现无状态登录与鉴权：客户端登录后获取 token，请求时在 Authorization Header 中携带 Bearer token，服务端过滤器解析并注入安全上下文，实现接口级权限控制。

## 2.3 Vue3 + Vite
Vue3 提供组件化开发模型与响应式数据绑定；Vite 提供快速构建与热更新。本文前端采用 Vue3 + Vite 实现单页应用，通过 Router 进行页面组织，通过 Axios 与后端交互。

## 2.4 MySQL 与 JPA
MySQL 作为常用关系型数据库，适合存储结构化业务数据。JPA 通过 ORM 映射实体与表结构，降低数据访问成本，并支持约束与事务。

## 2.5 可行性分析
技术可行性：所用技术栈成熟稳定；经济可行性：均为开源技术；运行可行性：提供 Docker Compose 一键部署，便于在本地与服务器环境运行。

# 3 需求分析
## 3.1 角色与权限
系统设定三类角色：
- ADMIN：用户列表、审计查询、系统管理
- MENTOR：创建课题组、管理成员、创建项目、审核周报、审计查询
- STUDENT：加入多个课题组、任务协作、提交周报、查看审核结果

## 3.2 功能需求
（1）用户登录认证：登录获取 JWT，访问业务接口需携带 token。
（2）多课题组管理：导师创建课题组；成员可被加入多个课题组；可查看组成员并增删。
（3）项目任务协作：按课题组创建项目；项目下管理任务；任务支持状态流转 TODO/DOING/DONE。
（4）周报提交与导师审核：学生按自然周提交周报；导师对周报审核通过或驳回并填写意见。
（5）审计与可追溯：对关键操作留痕，并提供查询。

## 3.3 非功能需求
- 安全性：JWT + 角色权限；成员关系校验
- 可用性：Docker 一键启动
- 可维护性：模块化、分层结构

# 5 系统详细设计与实现（结合仓库实际实现）
## 5.1 登录认证实现
后端提供登录接口：
- POST /api/auth/login
登录成功返回 token，前端保存在 localStorage，并在 Axios 请求拦截器中自动携带 `Authorization: Bearer <token>`。

后端通过 `JwtAuthFilter` 对除 `/api/auth/**` 等白名单外的请求进行 token 解析，并注入 SecurityContext，实现无状态鉴权。

## 5.2 多课题组管理实现
课题组模块包含：
- Group：课题组实体
- GroupMember：成员关系实体（group_id + user_id 唯一）

主要接口：
- GET /api/groups/my：查询当前用户加入的课题组
- POST /api/groups：创建课题组（MENTOR/ADMIN）
- GET /api/groups/{groupId}/members：查询成员
- POST /api/groups/{groupId}/members：添加成员
- DELETE /api/groups/{groupId}/members/{memberId}：移除成员（设置 inactive）

前端页面 `/groups` 提供创建组、成员管理功能。

## 5.3 项目与任务协作实现
项目接口：
- GET /api/projects/by-group/{groupId}
- POST /api/projects

任务接口：
- GET /api/tasks/by-project/{projectId}
- POST /api/tasks
- PUT /api/tasks/{taskId}

任务状态 TaskStatus：TODO、DOING、DONE。前端页面 `/projects` 以三列看板展示任务，并通过更新接口实现状态流转。

## 5.4 周报提交与导师审核实现
周报表 weekly_reports 采用自然周字段：weekYear、weekNumber；并对同一组同一作者同一自然周设置唯一约束（避免重复提交）。

接口：
- POST /api/weekly-reports/submit：提交周报
- GET /api/weekly-reports/my：我的周报列表
- GET /api/weekly-reports/by-group/{groupId}：按组查看周报
- POST /api/weekly-reports/{reportId}/review：导师审核（approve=true/false + mentorComment）

前端页面 `/weekly`：学生提交与查看结果；导师在同页加载组周报并进行审核。

## 5.5 审计与可追溯实现
系统引入 audit_logs 表，记录 actor、entityType、entityId、action、detail、at。

关键操作写入审计：
- 创建/修改课题组
- 成员添加/移除
- 创建/更新/删除任务
- 周报提交与审核

审计查询接口：
- GET /api/audit?actor=&entityType=&action=
前端页面 `/audit` 提供筛选查询。
