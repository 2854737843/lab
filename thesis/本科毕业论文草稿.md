# 本科毕业论文（草稿）

> 题目：基于 Spring Boot + Vue 的课题组项目协同与周报管理系统的设计与实现

## 摘要
随着高校科研团队规模扩大，导师与学生之间在课题组管理、任务协作、周报提交与审核等方面对信息化平台的需求日益增强。本文设计并实现了一套基于 Spring Boot 与 Vue 的课题组项目协同与周报管理系统。系统提供用户登录认证、多课题组管理、项目任务协作、周报提交与导师审核、审计与可追溯等功能。后端采用 Spring Boot 构建 REST API，使用 Spring Security + JWT 实现无状态认证授权，数据持久化采用 JPA + MySQL；前端采用 Vue3 + Vite 搭建单页应用。系统通过审计日志对关键业务操作进行记录，实现操作可追溯。测试结果表明，该系统能够满足课题组日常协作管理需求，提高协作效率并增强过程管理能力。

**关键词**：Spring Boot；Vue；JWT；课题组管理；周报管理；审计日志

## 1 绪论
### 1.1 研究背景与意义
（1）课题组协作问题：成员多、任务分散、沟通与过程记录困难。
（2）周报制度问题：缺少统一平台，提交、审核、统计成本高。
（3）可追溯需求：科研管理需要留痕记录，便于复盘与考核。

### 1.2 国内外研究现状
可简单描述常见协作工具（如 Trello、Jira、飞书、钉钉）与高校场景差异，强调定制化系统的必要性。

### 1.3 本文主要工作
- 完成系统需求分析与功能划分
- 完成总体架构设计与数据库设计
- 实现用户认证、多课题组、任务协作、周报与审核、审计追溯
- 完成系统测试与部署说明

## 3 需求分析
### 3.1 角色与权限
- 管理员（ADMIN）：系统配置、用户管理、审计查询
- 导师（MENTOR）：创建课题组、管理成员、项目与任务管理、周报审核、审计查询
- 学生（STUDENT）：参与多个课题组、提交周报、查看任务

### 3.2 功能需求
1. 用户登录认证
2. 多课题组管理（用户可加入多个课题组）
3. 项目任务协作（TODO/DOING/DONE）
4. 周报提交与导师审核（自然周）
5. 审计与可追溯（记录关键操作）

### 3.3 非功能需求
- 安全性：JWT + 角色权限；接口鉴权
- 可用性：Docker 一键部署
- 可维护性：分层设计、模块化
